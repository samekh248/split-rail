# Phase 1 Data Model: Complete Organization & Venue CRUD

This feature is additive to the existing tenant schema. Only one schema change is introduced (`organizations.archived_at`); all other entities are referenced for behavior/validation, not modified.

## Entities

### Organization (`organizations`) — MODIFIED

| Field | Type | Notes |
|-------|------|-------|
| `Id` | `Guid` | PK; `gen_random_uuid()` default |
| `Name` | `string` | Required; trimmed; **≤200 chars (app-layer)**; DB column `name varchar(255)` (backstop, unchanged) |
| `CreatedAt` | `DateTimeOffset` | `NOW()` default |
| **`ArchivedAt`** | **`DateTimeOffset?`** | **NEW.** `NULL` = active; non-null = archived (soft-deleted). Column `archived_at timestamptz NULL` |

Relationships (existing, unchanged): `Venues` (1‑*), `Roles` (1‑*), `UserMappings` (1‑*), `Invitations` (1‑*).

**State**:
- Active → Archived: set `ArchivedAt = now` via organization delete, **only when empty** (no venues and no events/financial line items). Irreversible within this feature's scope (no un-archive endpoint).
- Archived organizations are excluded from all reads via the tenant query filter; underlying child records are retained.

**Validation rules**:
- `Name`: required, not empty/whitespace after trim, length ≤ 200 (FR-010, FR-011).
- Delete allowed only when no owned venues and no financial data exist (FR-008); otherwise `ConflictException`.

### Venue (`venues`) — UNCHANGED (behavior added)

| Field | Type | Notes |
|-------|------|-------|
| `Id` | `Guid` | PK |
| `OrganizationId` | `Guid` | FK → organizations; tenant scope |
| `Name` | `string` | Required; trimmed; **≤200 chars (app-layer)**; DB `varchar(255)` |
| `CreatedAt` | `DateTimeOffset` | `NOW()` default |

**Validation rules**:
- `Name`: required, not empty/whitespace after trim, length ≤ 200 (FR-010, FR-011), applied on create and update.
- Update authorized only when caller holds `can_manage_permissions` AND the venue is within the caller's accessible scope (FR-003–FR-005).

### Organization Membership (`user_organization_mappings`) — UNCHANGED (used by list)

| Field | Type | Notes |
|-------|------|-------|
| `UserId` | `Guid` | PK part; FK → users |
| `OrganizationId` | `Guid` | PK part; FK → organizations |
| `RoleId` | `Guid` | FK → organization_roles |

Drives the organization list (the orgs a user belongs to) and, via `Role.CanManagePermissions`, the admin gate for org update/delete.

### OrganizationRole (`organization_roles`) — UNCHANGED (used for gating)

`CanManagePermissions` is the admin capability gating organization update/delete and venue update (no new permission type introduced).

### Financial data (referenced for delete-block) — UNCHANGED

- `Venue` (org owns venues) — presence blocks org delete.
- `Event` (`events`) / `FinancialLineItem` (`financial_line_items`) reached via venue → event chain — presence blocks org delete (financial-integrity protection, Constitution V).

## Migration

- **Name**: `AddOrganizationArchivedAt` (additive).
- **Up**: `ALTER TABLE organizations ADD COLUMN archived_at timestamptz NULL;`
- **Down**: drop the column.
- No data backfill required (`NULL` default = all existing orgs active). Update `ApplicationDbContextModelSnapshot` accordingly (generated by `dotnet ef migrations add`).

## Query-filter change

In `ApplicationDbContext.ApplyTenantQueryFilters`, extend the `Organization` filter to also exclude archived rows:

```csharp
modelBuilder.Entity<Organization>().HasQueryFilter(e =>
    (_tenantContext.OrganizationId == null || e.Id == _tenantContext.OrganizationId)
    && e.ArchivedAt == null);
```

- Effect: `GetCurrentOrganizationAsync` and any tenant-scoped org read automatically hide archived orgs (FR-007, edge: "Read excludes archived organizations").
- The organization **list** query uses `IgnoreQueryFilters()` and applies its own membership + `ArchivedAt == null` predicate (see research §3), so multi-org membership is respected while archived orgs stay hidden.
- The delete-block existence checks (venues/events/line items) run against tenant-filtered sets scoped to the caller's org.
