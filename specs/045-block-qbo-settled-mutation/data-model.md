# Data Model: QBO Sync Frozen-Event Guard

**Feature**: 045-block-qbo-settled-mutation  
**Date**: 2026-06-20

## Purpose

Document the settlement snapshot field inventory, sanctioned reconciliation fields, and lifecycle write matrix governing QuickBooks sync recompute — without schema changes.

## Entities (existing — no migrations)

### Event

| Field | Role in this feature |
|-------|---------------------|
| `Id` | Sync target identifier |
| `VenueId` | Audit log scope |
| `Status` | **`PRE_SHOW` \| `SETTLED` \| `RECONCILED`** — determines sync write permissions |
| `SettlementPdfUrl` | Must not be regenerated by sync (invariant) |

### FinancialLineItem

| Field | Category | Sync write on SETTLED | Sync write on RECONCILED |
|-------|----------|----------------------|--------------------------|
| `ProformaValue` | Settlement snapshot | **Blocked** | **Blocked** |
| `SettlementValue` | Settlement snapshot | **Blocked** | **Blocked** |
| `QboActualValue` | Sanctioned reconciliation | **Blocked** | **Permitted** |
| `BlockType` | Settlement snapshot metadata | **Blocked** | **Blocked** |
| `RowLabel` | Settlement snapshot metadata | **Blocked** | **Blocked** |
| `SortOrder` | Settlement snapshot metadata | **Blocked** | **Blocked** |
| `IsArtistDeduction` | Settlement snapshot metadata | **Blocked** | **Blocked** |
| `Notes` | Settlement snapshot metadata | **Blocked** | **Blocked** |
| `IsHiddenFromPromoter` | Settlement snapshot metadata | **Blocked** | **Blocked** |
| `UpdatedAt` | Audit timestamp | **Blocked** (with line item) | **Permitted** (when actuals change) |

**Settlement snapshot definition**: Fields captured in `SettlementLineItemSnapshot` at finalize (`BlockType`, `RowLabel`, `SortOrder`, `IsArtistDeduction`, `SettlementValue`) plus `ProformaValue` used in pre-settlement grid. These map 1:1 to PDF snapshot content.

**Sanctioned reconciliation fields**: `QboActualValue` and companion `UpdatedAt` when only actuals change. Variance is derived at read time (`QboActualValue - SettlementValue`) — not persisted.

### QboSyncLedger (unchanged — ingest continues)

Append-only cache of ingested QBO transactions. Not subject to frozen-event line-item guard; governed by Constitution IV append-only rules separately.

## State transitions (relevant subset)

```text
PRE_SHOW ──finalize──► SETTLED ──reconcile──► RECONCILED
   │                      │                      │
   │ sync: full writes    │ sync: cache only;    │ sync: cache + actuals
   │                      │ recompute REJECTED   │ recompute ALLOWED
   └──────────────────────┴──────────────────────┘
              settlement snapshot immutable after finalize
```

## Validation rules

1. **VR-001**: Before any `financial_line_items` mutation in sync recompute, load parent event status.
2. **VR-002**: If status is `SETTLED` and recompute would modify any line item → reject with `LedgerStateException`, audit `qbo_sync_recompute`.
3. **VR-003**: If status is `RECONCILED`, recompute may set only `QboActualValue` and `UpdatedAt` per line item.
4. **VR-004**: Persistence interceptor must enforce VR-003 and reject VR-002 violations even if service guard is bypassed.
5. **VR-005**: `events` and `event_artists` mutations during sync remain prohibited on frozen events (existing interceptor coverage).

## Invariants

- **INV-001**: After finalize, `SettlementValue` and `ProformaValue` on all line items are byte-stable across sync operations.
- **INV-002**: Archived settlement PDF bytes unchanged after any sync on frozen events (including successful RECONCILED actuals refresh).
- **INV-003**: Rejected sync attempts emit exactly one audit log per rejection (spec 039 contract).

|-------|------------------|
| `qbo_sync_recompute` | Sync recompute attempted on `SETTLED` event, or non-actuals field change during sync on frozen event |
